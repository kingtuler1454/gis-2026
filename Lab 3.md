# Лабораторная работа №3
## «Совместный анализ данных с Overture Maps и построение тематической карты»

### 1. Цель работы
Освоить методы интеграции локальных пространственных данных с облачными наборами Overture Maps, выполнить пространственный анализ в DuckDB, сформировать итоговый GeoJSON‑слой и визуализировать его в веб‑клиенте, созданном в рамках ЛР2, с использованием Mapbox Style.

---

## 2. Исходные данные
1. GeoJSON‑файл, полученный студентом в результате выполнения Лабораторной работы №1 (оцифрованные здания).
2. Локальная база данных DuckDB.
3. Данные Overture Maps Buildings (GeoParquet), доступные по HTTP(S).
4. Веб‑клиент, разработанный в Лабораторной работе №2 (OpenLayers + Mapbox Style).

---

## 3. Содержание задания

### 3.1. Загрузка пользовательского GeoJSON в DuckDB
1. Создать локальную базу данных DuckDB.
2. Загрузить входной GeoJSON‑файл в таблицу базы данных.
3. Убедиться, что геометрия корректно приведена к типу GEOMETRY.

### 3.2. Подключение данных Overture Maps
1. Выполнить подключение GeoParquet‑файлов Overture Maps на уровне SQL‑запросов, без предварительного скачивания.
2. Определить список партиций Overture Maps, пересекающихся с bbox.
3. Работа должна осуществляться только с необходимыми партициями. Данные можно сохранить в таблицы базы данных.

Ссылки:
- Overture Maps: https://overturemaps.org
- STAC API Landing Page: https://stac.overturemaps.org/

### 3.3.  Пространственное пересечение с пользовательскими данными
Добавить в таблицу с данными Overture Maps поле, отражающее источник данных:

**my** — если объект пересекается с геометрией из таблицы, полученной на шаге 3.1;

**osm** — если в Overture Maps указано происхождение OSM;

**ml** — если объект получен автоматическим распознаванием (например, Microsoft/Google/ML‑pipeline).

Поле можно назвать source_type.

### 3.4. Формирование итогового GeoJSON‑слоя
На основе таблицы overture_buildings сформировать итоговый GeoJSON‑файл overture.geojson.
В GeoJSON должны быть:
- геометрия,
- поле source_type,
- любые дополнительные атрибуты по вашему усмотрению.

### 3.5. Визуализация в клиенте (ЛР2)
1. Подключить итоговый GeoJSON‑слой напрямую (без GeoServer).
2. Настроить стилизацию с помощью Mapbox Style.
3. Использовать библиотеку ol-mapbox-style для применения стилей.
3. Построить хороплет, основанный на поле source_type.

Примерная цветовая схема:

my — зелёный

osm — синий

ml — оранжевый

- SQL‑скрипт, содержащий все этапы обработки данных
